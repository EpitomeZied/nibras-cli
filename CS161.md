## Grading

Each project uses `check` grading. For instructor-only grading, keep
`grading.json` in a private grading repo and set `NIBRAS_GRADING_ROOT` (or pass
`--grading-root`). `nibras` will auto-check answers using one file per question
(e.g., `q1.txt`). When `requireGrading` is enabled (default), `grading.json` is
required and manual scoring is disabled. You can store answers in a different
folder by passing `--answers-dir`.

For `exam1`, the repo includes sample answer files under
`sample-answers/cs161/exam1/`. Use them to validate your private grading rules:

```bash
nibras cs161 test exam1 --grading-root /path/to/private/grading --answers-dir sample-answers/cs161/exam1
```

Expected private grading layout:

```text
<grading-root>/cs161/exam1/grading.json
```

`exam1` answer-file contract:
- `q1.txt`
- `q2.txt`
- `q3.txt`

Auto-check example (`grading.json`, strict exact match):

```json
{
  "totalPoints": 100,
  "questions": [
    {
      "id": "q1",
      "points": 45,
      "answerFile": "q1.txt",
      "solutions": ["Answer A", "Answer B"]
    }
  ]
}
```

Auto-check rules:
- Trim and collapse whitespace.
- Case-sensitive exact match.
- No partial credit.

Validation rules:
- `grading.json` must exist when `requireGrading` is enabled or when `NIBRAS_GRADING_ROOT`
  or `--grading-root` is set.
- Question IDs must be unique.
- Sum of question points must equal `totalPoints`.
- Each `answerFile` must exist and be non-empty.

## Commands

nibras cs161 test exam1
nibras cs161 test exam1 --answers-dir /path/to/my/answers
nibras cs161 test exam1 --grading-root /path/to/private/grading --answers-dir /path/to/my/answers
nibras cs161 test section4
nibras cs161 submit exam2
